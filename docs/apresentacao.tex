\begin{frame}{Modelagem: Grafo Bipartido}
    O sistema inicial é modelado como um grafo bipartido $G = (P, I, E)$:
    \begin{description}
        \item[Vértices Pratos ($P$):] Representam os itens finais do cardápio.
        \item[Vértices Ingredientes ($I$):] Representam os componentes atômicos.
        \item[Arestas ($E$):] Uma conexão binária indicando que um ingrediente compõe o prato.
    \end{description}
    \vspace{0.3cm}
    \textbf{Vantagem Teórica:} A estrutura bipartida impede conexões diretas entre pratos, forçando a análise da vizinhança comum para determinar similaridade técnica.
\end{frame}

\begin{frame}{Métrica: Índice de Jaccard}
    A similaridade entre dois pratos $u$ e $v$ é quantificada matematicamente para a geração das projeções:
    \[ J(u,v) = \frac{|N(u) \cap N(v)|}{|N(u) \cup N(v)|} \]
    \begin{itemize}
        \item $N(u) \cap N(v)$: Ingredientes compartilhados (Interseção).
        \item $N(u) \cup N(v)$: Repertório total de ingredientes (União).
    \end{itemize}
    \textbf{Aplicação:} Identificar o "espelhamento" de receitas entre os restaurantes.
\end{frame}
